1. Overview & Requirements

What is Digambar 11?

A private fantasy IPL app for friend groups. No real money involved — players pay ₹25 virtual currency per match, independently pick 11 players from both teams' full squads (Dream11-style, not draft), and compete on a season-long leaderboard. The app opens 24 hours before each match. Players can be picked from the full squad — if a player doesn't make the Playing XI, they score 0 points. Dynamic credits adjust after each match based on performance.

Why everything on one EC2? With ~50 concurrent users, there's no need for microservices. A single t3.micro (or t2.micro free tier) running Node.js, PostgreSQL, Redis, and the scrapers handles this load easily. This keeps complexity and cost near zero.

Scraper Resilience Strategy

Since scrapers can break if the source website changes its HTML structure, the system uses a cascading fallback approach:

  1. Primary: sanwebinfo/cricket-api scraper (Cricbuzz)
  2. Fallback 1: rsumit123/ipl_live scraper (also Cricbuzz, different parsing logic)
  3. Fallback 2: CricAPI free tier (100 req/day — enough for ~6 matches if scrapers fail)
  4. Manual Override: Admin endpoint to manually push score updates if all else fails

6. Real-Time Architecture

LIVE MATCH FLOW (every 15 seconds) ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ Cricbuzz ──scrape──► Score Scraper ──► Redis Cache │ ┌────────┤ │ │ ┌──────▼─────┐ │ │ Scoring │ │ │ Engine │ │ │ │ │ │ For each │ │ │ player in │ │ │ match: │ │ │ calculate │ │ │ fantasy │ │ │ points │ │ └──────┬─────┘ │ │ │ ┌──────▼─────┐ │ │ PostgreSQL │ │ │ UPDATE │ │ │ scores │ │ └──────┬─────┘ │ │ │ ┌──────▼─────┐ │ │ Redis │◄─┘ │ PUBLISH │ │ channel: │ │ match:14 │ └──────┬─────┘ │ ┌──────▼─────────────────────┐ │ Socket.IO Server │ │ SUBSCRIBE → broadcast to │ │ all connected clients in │ │ rooms: match:14:league:X │ └─────────────────────────────┘

Redis Pub/Sub decouples the scraper from the WebSocket server. The scraper publishes score updates to a channel, and Socket.IO subscribes and broadcasts to the right rooms. This means if you ever need to scale to multiple Node processes, they all subscribe to the same Redis channel.

Key Design Decision: Players pick from the FULL SQUAD, not the Playing XI. The Playing XI isn't known until 30 minutes before the match (at toss). This means users must predict who'll play — adding a risk/reward element. A player who's been in amazing form but gets dropped = 0 points for anyone who picked them.

Dynamic Credit System

Base Credit Formula (Start of Season)

base_credit = 6.0 + 5.0 × (auction_price / max_auction_price) ^ 0.6

// Examples (max = ₹27 Cr — Rishabh Pant):
// ₹27.00 Cr → 11.0 credits  (Rishabh Pant)
// ₹21.00 Cr → 10.4 credits  (Virat Kohli)
// ₹18.00 Cr → 10.0 credits  (Bumrah, Jaiswal)
// ₹10.00 Cr →  9.1 credits  (Mohammed Shami)
// ₹5.00 Cr  →  8.0 credits  (Rinku-tier)
// ₹0.30 Cr  →  6.1 credits  (uncapped players)

After Each Match — Credit Adjustment

expected_pts = current_credit × 10  // 9.0 credit → expect ~90 pts
performance_factor = actual_fantasy_pts / expected_pts

if (performance_factor > 1.5)  → credit += 1.0  // Exceptional
if (performance_factor > 1.2)  → credit += 0.5  // Good
if (performance_factor < 0.5)  → credit -= 0.5  // Poor
if (player didn't play)        → credit -= 0.5  // Dropped
otherwise                      → no change       // As expected

// Guardrails
MIN_CREDIT = 6.0   // Floor
MAX_CREDIT = 12.0  // Ceiling
MAX_CHANGE = 1.0   // Per match

Team Validation (Server-Side)

function validateTeam(selectedPlayers, matchId) {
  const errors = [];

  // Exactly 11 players
  if (selectedPlayers.length !== 11)
    errors.push('Must pick exactly 11 players');

  // Credit cap
  const totalCredits = selectedPlayers.reduce((s, p) => s + p.current_credit, 0);
  if (totalCredits > 100)
    errors.push(`Over credit cap: ${totalCredits}/100`);

  // Max 4 overseas
  const overseas = selectedPlayers.filter(p => p.nationality === 'Overseas');
  if (overseas.length > 4)
    errors.push(`Max 4 overseas, you have ${overseas.length}`);

  // Role minimums
  const roles = { WK: 0, BAT: 0, BWL: 0, AR: 0 };
  selectedPlayers.forEach(p => roles[p.role]++);
  if (roles.WK < 1) errors.push('Need at least 1 WK');
  if (roles.BAT < 3) errors.push('Need at least 3 BAT');
  if (roles.BWL < 3) errors.push('Need at least 3 BWL');
  if (roles.AR < 1) errors.push('Need at least 1 AR');

  // Role maximums
  if (roles.WK > 4) errors.push('Max 4 WK');
  if (roles.BAT > 6) errors.push('Max 6 BAT');
  if (roles.BWL > 6) errors.push('Max 6 BWL');
  if (roles.AR > 4) errors.push('Max 4 AR');

  // All from match squads
  // ... verify all players belong to the two teams playing

  return { valid: errors.length === 0, errors };
}

8. Scoring Engine

// scoring-engine.js — Fantasy Points Calculator

const SCORING_RULES = {
  batting: {
    run:              1,
    four_bonus:       1,    // per boundary
    six_bonus:        2,    // per six
    half_century:     8,    // 50 runs bonus
    century:          16,   // 100 runs bonus
    duck:            -2,    // 0 runs (BAT/WK/AR only)
    sr_bonus_170:     6,    // SR > 170 (min 10 balls)
    sr_bonus_150:     4,    // SR 150-170
    sr_penalty_60:   -6,    // SR < 60 (min 10 balls)
    sr_penalty_70:   -4,    // SR 60-70
  },
  bowling: {
    wicket:          25,
    three_wicket:     4,    // 3W haul bonus
    five_wicket:      8,    // 5W haul bonus
    maiden:          12,
    eco_below_5:      6,    // econ < 5 (min 2 overs)
    eco_below_6:      4,    // econ 5-6
    eco_above_10:    -6,    // econ > 10
    eco_above_9:     -4,    // econ 9-10
  },
  fielding: {
    catch:            8,
    stumping:        12,
    run_out_direct:  12,
    run_out_indirect: 6,
  },
  multipliers: {
    captain:          2.0,
    vice_captain:     1.5,
  }
};

function calculatePlayerPoints(stats, role) {
  let pts = 0;

  // Batting
  pts += stats.runs * SCORING_RULES.batting.run;
  pts += stats.fours * SCORING_RULES.batting.four_bonus;
  pts += stats.sixes * SCORING_RULES.batting.six_bonus;
  if (stats.runs >= 100) pts += SCORING_RULES.batting.century;
  else if (stats.runs >= 50) pts += SCORING_RULES.batting.half_century;
  if (stats.runs === 0 && stats.balls_faced > 0
      && ['BAT','WK','AR'].includes(role)) {
    pts += SCORING_RULES.batting.duck;
  }

  // Strike rate bonus/penalty (min 10 balls)
  if (stats.balls_faced >= 10) {
    const sr = (stats.runs / stats.balls_faced) * 100;
    if (sr > 170)     pts += SCORING_RULES.batting.sr_bonus_170;
    else if (sr > 150) pts += SCORING_RULES.batting.sr_bonus_150;
    else if (sr < 60)  pts += SCORING_RULES.batting.sr_penalty_60;
    else if (sr < 70)  pts += SCORING_RULES.batting.sr_penalty_70;
  }

  // Bowling
  pts += stats.wickets * SCORING_RULES.bowling.wicket;
  if (stats.wickets >= 5) pts += SCORING_RULES.bowling.five_wicket;
  else if (stats.wickets >= 3) pts += SCORING_RULES.bowling.three_wicket;
  pts += stats.maidens * SCORING_RULES.bowling.maiden;

  // Economy bonus/penalty (min 2 overs)
  if (stats.overs >= 2) {
    const eco = stats.runs_given / stats.overs;
    if (eco < 5)       pts += SCORING_RULES.bowling.eco_below_5;
    else if (eco < 6)  pts += SCORING_RULES.bowling.eco_below_6;
    else if (eco > 10) pts += SCORING_RULES.bowling.eco_above_10;
    else if (eco > 9)  pts += SCORING_RULES.bowling.eco_above_9;
  }

  // Fielding
  pts += stats.catches * SCORING_RULES.fielding.catch;
  pts += stats.stumpings * SCORING_RULES.fielding.stumping;
  pts += stats.run_outs * SCORING_RULES.fielding.run_out_direct;

  return pts;
}

11. Deployment & DevOps

EC2 Setup Script

#!/bin/bash
# Setup Digambar 11 on Ubuntu 22.04 EC2

# System
sudo apt update && sudo apt upgrade -y
sudo apt install -y nginx postgresql redis-server python3-pip nodejs npm

# PostgreSQL
sudo -u postgres createdb digambar11
sudo -u postgres psql -c "CREATE USER dg11 WITH PASSWORD 'your_password';"
sudo -u postgres psql -c "GRANT ALL ON DATABASE digambar11 TO dg11;"

# Redis
sudo systemctl enable redis-server

# Score Scrapers
cd /opt
sudo git clone https://github.com/sanwebinfo/cricket-api.git
sudo git clone https://github.com/rsumit123/ipl_live.git
cd cricket-api && pip3 install -r requirements.txt
cd ../ipl_live && pip3 install -r requirements.txt

# App
cd /opt
git clone https://github.com/piyush-tkd/digambar11.git
cd digambar11
npm install
npm run build

# PM2 (process manager)
npm install -g pm2
pm2 start ecosystem.config.js
pm2 startup
pm2 save

# Nginx
sudo cp nginx.conf /etc/nginx/sites-available/digambar11
sudo ln -s /etc/nginx/sites-available/digambar11 /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx

# SSL via Cloudflare (point DNS, enable Full SSL)
